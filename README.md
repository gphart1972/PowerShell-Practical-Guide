# PowerShell User Guide

Welcome to the **PowerShell User Guide**, a curated collection of practical, real‑world scripts designed to automate common system‑administration, file‑management, and workflow tasks.  
Each section in the `/docs` directory focuses on a specific problem area and includes:

- A clear description of the task
- Usage instructions
- Sanitized, ready‑to‑run PowerShell scripts
- Notes, tips, and optional enhancements

This guide is intended for beginners and intermediate PowerShell users who want to build a reliable toolbox of automation utilities.

---

## 📁 Directory Structure

The `/docs` folder contains the following script categories:

- **[Cleanup Empty Dirs](ca://s?q=Open_Cleanup_Empty_Dirs_section)** — Identify and remove empty directories safely.
- **[Cleanup-Installer Orphans](ca://s?q=Open_Cleanup_Installer_Orphans_section)** — Locate leftover installer files and temporary artifacts.
- **[Delete duplicate files](ca://s?q=Open_Delete_Duplicate_Files_section)** — Scan for and remove duplicate files using hashing.
- **[Domain Join Automation](ca://s?q=Open_Domain_Join_Automation_section)** — Automate joining Windows machines to a domain.
- **[Filename List](ca://s?q=Open_Filename_List_section)** — Generate plain-text filename lists from any folder.
- **[Find long names](ca://s?q=Open_Find_Long_Names_section)** — Detect files or folders exceeding Windows path-length limits.
- **[Linux Stuff](ca://s?q=Open_Linux_Stuff_section)** — Cross‑platform scripts and notes for PowerShell on Linux.
- **[Move from folders](ca://s?q=Open_Move_From_Folders_section)** — Flatten directory structures by moving files upward.
- **[Windows Batch Scripts (Pre-PowerShell Era)](ca://s?q=Open_Windows_Batch_Scripts_section)** — Legacy `.bat` scripts preserved for reference.
- **[remove old files](ca://s?q=Open_Remove_Old_Files_section)** — Automatically delete files older than a specified age.

Each section includes Markdown documentation and one or more scripts.

## 🧭 How to Use This Guide

1. Browse to the section that matches the task you want to automate.  
2. Review the description and usage notes.  
3. Copy the script into your environment or run it directly in PowerShell 5.1 or PowerShell 7+.  
4. Modify paths and variables as needed for your system.  
5. Explore related sections to build more advanced workflows.

If you're new to PowerShell, start with:

- **[Basic file operations](ca://s?q=Explain_PowerShell_file_operations)**  
- **[Understanding the pipeline](ca://s?q=Explain_PowerShell_pipeline)**  
- **[Working with objects](ca://s?q=Explain_PowerShell_objects)**
